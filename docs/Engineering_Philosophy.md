# Engineering Philosophy

DevPit is an engineering attention center, not a Git dashboard or project
management tool. It answers one question: **what requires my attention now?**

## Guiding principles

- **User-centric** — the engineer is the center of the workflow, not the
  repository.
- **Attention over information** — surface actionable work, not every
  notification.
- **Provider-agnostic** — providers are peers, integrated as plugins.
- **Self-host first** — a single binary or container the user owns.
- **Read-only by default** — aggregate; actions stay in the source systems.
- **Extensible** — a plugin architecture for new providers.
- **Fast and keyboard-first** — the UI is operated at speed.
- **Progressive disclosure** — the list shows the minimum to act on; detail
  appears on demand (hover text).

This is the canonical list; other docs (e.g. `docs/Why.md`) link here rather
than restating it.
